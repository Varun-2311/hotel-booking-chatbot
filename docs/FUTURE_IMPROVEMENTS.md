# Challenges And Future Improvements

## Challenges

- Preventing the LLM from inventing missing booking fields
- Keeping retrieval lightweight for a simple Streamlit deployment
- Handling email failures without losing the saved booking
- Preserving conversational continuity without repetitive replies

## Future Improvements

- Add STT/TTS support
- Add true room inventory and availability logic
- Persist the vector index across restarts
- Add admin authentication
- Support full modify-booking flows in chat
- Add richer evaluation datasets and regression tests
- Deploy publicly and capture production screenshots
